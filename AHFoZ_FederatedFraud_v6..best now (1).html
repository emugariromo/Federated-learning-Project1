Privacy Architecture & Compliance
✅ Raw data never leaves the client node. Only model weight summaries and anonymised alert metadata are transmitted to the central server.
✅ FedAvg (McMahan et al. 2017) — averages model insights across all nodes without any individual claims data being shared.
✅ Compliant with Zimbabwe Data Protection Act and AHFoZ data governance standards.
✅ Cross-funder intelligence uses only provider IDs and claim counts — no member demographics, no clinical data, no amounts transmitted.
✅ This system implements the research proposed in AHFoZ Topic Proposal v2: Federated Time-Series Modelling for Collaborative Healthcare Fraud Detection.

Share with Client Nodes
The Python server is the bridge. Once it's running, any machine on your network can submit results directly into your SQLite database.
Step-by-step:
1. Run python ahfoz_server.py in VS Code terminal
2. Terminal shows: NODES: http://192.168.x.x:5000
3. Copy that URL and send it via WhatsApp / email to each node
4. Node opens URL in their browser → uploads CSV → Run → Submit
5. You click ☁ Sync on Central Server → all their results appear
All data goes into SQLite (ahfoz_data.db) on your machine. No internet needed.
